부트스트랩 치트시트 by 안상봉

머리말 
                        부트스트랩 4.6 버전 기준으로 작성. 
                        DM9도 부트스트랩 4 기반임
                        핵심적인 내용은 다른 버전과 큰차이 없음
                        주로 쓰는건 3,4 이며, 최근 클래스이름 약간 변경+애니메이션 빵빵한 5가 나왔으나,
                        부트스트랩은 갈수록 안쓰는 추세. but 배우기 쉽고 이미 부트스트랩으로 구축된 사이트가 매우매우 많고
                        입문용으로 최고인거 같음

                    1. 레이아웃
                        - 기본 구성
                            - body
                            - header
                                - nav
                            - main
                                - section
                                - arcicle

                        - 시맨틱 태그 활용한 메뉴 구성 + 안에 콘텐츠 집어넣고 콘텐츠 삽입
                            - 구성하기 나름이며, 꼭 저대로 안해도 됨
                            - 쟤네들 안에 .container 넣고 그리드 적용한 div로 컨텐츠를 배치함
                            - 기본적으로 부트스트랩4는 flex 기반으로 레이아웃이 작성되었으나, 다른것도 쓸수는 있음
        
                        - 그리드 시스템 (css grid 랑은 다르다)
                            - container
                                - 모든 부트스트랩 요소가 담기는 기본 컨테이너
                                - 보통 div.container 태그를 만들어 이걸로 싼(wrap)다
                                - 두종류가 있음.
                                    - 가로 마진 적당히 조절되는 container
                                        - 미디어쿼리 중단점마다 max-width 가 변함
                                    - 뷰포트에 무조건 꽉차는 fluid-container
        
                        - xs , sm , md , lg, xl, fluid
                            - 디바이스의 사이즈를 의미함.
                            - bs4는 모바일 퍼스트. 사이즈 미지정시 768px 미만 디바이스에서 운용됨을 기본가정으로 함
                            - 미디어쿼리로 중단점이 매겨져 있음
                            - 클래스이름에 얘네들이 들어가 있으면, 뷰포트 width 가 중단점 도달시 해당 클래스의 사이즈가 적용되
        
                        - 12 컬럼 베이스
                            - 태그에 클래스를 매겨 간격을 설정함
                            - 총합이 무조건 12일 필요는 없다
                            - 특정 컨테이너에 꽉차게 하고 싶으면  div.col-6, div.col-6 이거 두개
                            - 클래스 이름을 (레이아웃구성-디바이스사이즈-컬럼사이즈)
                            - 오프셋 → 무쓸모

                            클리어픽스란? 
                                부모요소가 자식요소를 감싸고 있을 때, 
                                자식에게 float 적용하면
                                부모요소가 자식요소를 감싸지 못하고, 
                                높이값을 파악하지 못하는 버그가 발생
                                이 float를 초기화(clear)하여 버그를 고쳐(fix)주는게 clearfix
                                clear:fix 스타일 적용하여 해결하나, bs4에서 기본기능 제공중.

                        - spacing
                            [] : 선택
                            {property}{side}-[{breakpoint}-]{size}
                            property
                                m 마진
                                p 패딩
                            side
                                {empty} 4방향
                                t top
                                b bottom
                                l left
                                r right
                                x 좌우
                                y 상하
                            size
                                $spacer : 디폴트로 1rem
                                sass 없으면 그냥 1rem
                                0 - for classes that eliminate the margin or padding by setting it to 0
                                1 - (by default) for classes that set the margin or padding to $spacer * .25
                                2 - (by default) for classes that set the margin or padding to $spacer * .5
                                3 - (by default) for classes that set the margin or padding to $spacer
                                4 - (by default) for classes that set the margin or padding to $spacer * 1.5
                                5 - (by default) for classes that set the margin or padding to $spacer * 3
                                auto - for classes that set the margin to auto
